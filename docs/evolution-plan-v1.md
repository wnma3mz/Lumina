# Lumina 1.0 进化计划 (Evolution Plan)

本计划旨在将 Lumina 从一个基础的本地 AI 工具箱，升级为一个能够「主动感知」用户活动、支持「实时系统音频翻译」并具备「开放扩展能力」的智能中枢。

---

## 核心任务 A：智能文件活动感知 (Pillar 2 - 智能摘要)
**目标**：让今日回顾不仅仅记录你在做什么，还要记录你「接触了什么」。

### 实施细节：
1. **文件活跃嗅探**：
   - 自动扫描 `~/Downloads`、`~/Desktop` 目录。
   - 依据 `ctime` (创建时间) 识别过去 24 小时内新增的文件。
2. **轻量级异步摘要**：
   - **PDF 摘要**：提取前 500 字符，调用本地 LLM 进行 30 字以内的极简摘要。
   - **代码/文本摘要**：识别语言，摘要核心功能。
   - **多媒体处理**：记录文件名与体积；针对图片，尝试进行基础 OCR。
3. **性能策略**：
   - 仅对 < 10MB 的文本类文件进行内容读取。
   - 超过阈值则仅展示文件名及来源（如「从浏览器下载」）。
4. **Digest 集成**：
   - 新增 `## 今日新增文件` 板块。

---

## 核心任务 B：系统级「本地同传」 (Pillar 4 - ASR 增强)
**目标**：实现在看生肉视频或在线会议时的实时翻译字幕。

### 实施细节：
1. **音频回路捕获 (Loopback)**：
   - 集成 `BlackHole` (macOS) 或 `Virtual Audio Cable` 驱动。
   - 实现「系统声音」作为 ASR 输入源的功能开关。
2. **实时字幕流**：
   - **流式转写**：使用 `mlx-whisper` / `faster-whisper` 的流式模式。
   - **AI 二次润色**：Whisper 结果 -> LLM 翻译并修正语气。
3. **展示层**：
   - 实现一个极简、透明、置顶的「桌面字幕条」（使用 `pywebview`）。

---

## 核心任务 C：Lumina 扩展协议 (Pillar 5 - 插件化)
**目标**：解耦核心逻辑，允许第三方开发者（或用户自己）编写插件。

### 实施细节：
1. **Collector 插件接口**：
   - 定义 `BaseCollector` 基类，支持自定义 `collect()` 方法。
   - 支持从 `~/.lumina/plugins/` 动态加载 `.py` 脚本。
2. **Action 动作钩子**：
   - 实现 `on_task_success` 钩子。
   - *应用场景*：处理完 PDF 后自动发送 Slack 通知、自动整理文件到指定文件夹。
3. **RPC 访问层**：
   - 提供更简洁的本地 Python API，让外部脚本调用 Lumina 的模型能力。

## 验收标准
1. **隐私性**：所有摘要和转写必须在本地完成，严禁联网。
2. **零干扰**：后台文件摘要不应占用过多 CPU 导致系统卡顿。
3. **易用性**：普通用户只需在 UI 勾选即可开启相应功能。
